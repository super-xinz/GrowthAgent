# ThreadPilot 多产品导航与管理设计

## 目标

让用户在任何页面都能直观地切换或新建产品，消除侧栏与内容区重复入口，并让总览真正反映多个产品，而不是隐式选取列表中的第一个产品。

## 信息架构

侧栏分为三层：品牌、全局产品切换器、当前上下文导航。全局导航只保留“总览”；产品上下文导航在当前 URL 包含产品 ID 时显示“产品概览、机会、对话、安全”。“添加产品”不再作为常驻导航项，而是产品切换器下拉列表中的固定末项“＋ 新建产品”。

产品切换器始终可见。未进入具体产品时显示“选择产品”；进入产品页面后显示当前产品名称与状态。展开后提供产品名称搜索、全部匹配产品，以及新建产品入口。选择产品默认进入其产品概览页。产品数量为零时直接展示空状态和新建入口。

## 总览与多产品管理

Dashboard 改为全局总览：顶部展示产品总数、分析完成数、运行中产品数和需要处理的产品数。下方使用产品卡片列表展示名称、状态、来源、候选社区数量和每日回复上限，并提供单一主入口“打开产品”。新建产品入口只在页面标题区保留一个，作为切换器以外的显式空状态/管理入口。

Dashboard 不再读取第一个产品的 analytics，也不再展示容易被误解为全局数据的扫描、机会、对话和漏斗数字。产品级数据继续留在对应产品页面。

## 产品页面

产品概览、机会、对话和安全页面删除与侧栏重复的内部跳转按钮。页面标题上方增加轻量面包屑“产品 / 当前产品 / 当前页面”，只承担定位，不重复制造操作入口。

产品概览页保留真正的操作：重新分析、启动或暂停影子模式、生成本地社区候选和产品设置。机会页保留候选处理动作；对话页保留模拟跟进和停止；安全页只展示风险与审计信息。

## 组件边界与数据流

- 根布局在服务端获取产品列表并传给客户端侧栏，避免每次切换器打开时额外请求。
- `ProductSwitcher` 只负责展开、搜索、选择产品和跳转新建页，不负责业务数据修改。
- `SidebarNav` 只根据 pathname 计算当前产品和当前导航高亮。
- `Breadcrumbs` 只根据当前产品与页面类型渲染定位信息。
- Dashboard 只使用产品列表做全局状态汇总，不混入产品级 analytics。

产品列表获取失败时沿用全局错误页与重试机制。切换器搜索无结果时显示“没有匹配的产品”，但始终保留“＋ 新建产品”。

## 响应式行为

桌面端侧栏固定，产品切换器使用侧栏内弹出列表。窄屏下侧栏变为页面顶部，切换器占满一行，导航横向滚动；弹出列表不超出视口。产品卡片在桌面端两列或三列，窄屏单列。

## 可访问性

切换器按钮提供 `aria-expanded` 和 `aria-controls`；下拉列表可通过 Escape 关闭，搜索框有明确标签；当前导航继续使用 `aria-current="page"`。状态不能只靠颜色表达，必须保留中文文本。

## 测试与验收

- 单元测试覆盖当前产品 ID 解析、产品搜索、产品状态汇总和导航高亮规则。
- 类型检查、ESLint、Next.js 生产构建和现有后端测试全部通过。
- 浏览器回归覆盖：总览多产品列表、切换器搜索与切换、切换器中新建产品、产品四个子页面高亮、重复按钮已删除、窄屏布局。
- 所有安全边界保持不变：Reddit 未批准时不进行真实抓取或发布，影子模式仍为默认模式。
