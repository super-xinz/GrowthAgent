# ThreadPilot 小红书评论增长改造设计

## 目标

将 ThreadPilot 从 Reddit 增长工具完整改造为小红书评论增长工具。系统通过本地 `xiaohongshu-mcp` 扫码登录、搜索笔记、读取笔记及评论、识别相关机会、生成评论草稿，并且只在用户逐条人工确认后发表一级评论或回复指定评论。

系统不发布小红书笔记，不上传图片或视频，不自动点赞或收藏，不批量自动评论；用户界面不再出现 Reddit、subreddit、影子模式或平台审批等术语。

## 基础设施与连接

Docker Compose 增加 `xiaohongshu-mcp` 服务，使用项目附带实现对应的预构建镜像，监听 `18060`。浏览器配置、Cookie 和运行数据保存到项目本地持久化目录，发布媒体目录不启用。

ThreadPilot API 增加独立的小红书客户端，调用 MCP 提供的 HTTP REST API：健康检查、登录状态、登录二维码、清理 Cookie、搜索笔记、笔记详情、当前账号、发表评论和回复评论。客户端统一设置连接与读取超时，将 MCP 错误转换为稳定的业务错误，绝不把 Cookie 返回给前端或日志。

## 登录流程

Dashboard 和独立账号页显示小红书连接状态。未登录时请求二维码并以图片展示，明确倒计时和刷新入口；用户使用小红书 App 扫码后，页面轮询登录状态。登录后显示当前账号名称和重新登录入口。重新登录必须经过确认，然后调用清理 Cookie 接口。

系统不保存小红书用户名或密码。Cookie 只保存在 MCP 服务的本地持久化目录。

## 平台数据模型

保留产品、Product Brain、查询词、候选机会、策略决策、回复草稿、发布记录、对话和审计的核心模型，但将平台字段泛化：

- 内容平台固定为 `XIAOHONGSHU`。
- 外部内容 ID 保存笔记 `feed_id`，同时保存 `xsec_token`。
- 内容类型包括 `NOTE` 和 `COMMENT`。
- subreddit 字段迁移为平台频道/来源标签，前端统一显示“小红书”。
- 评论机会额外保存目标评论 ID、目标用户 ID 和目标用户昵称。
- 发布记录保存操作类型 `COMMENT` 或 `COMMENT_REPLY`、平台返回结果和人工确认人/时间。

现有 Reddit 测试数据可以保留在数据库作为历史数据，但不再出现在新的小红书 UI；新数据全部标记小红书平台。

## 搜索与机会发现

Product Brain 根据目标用户、痛点、使用场景和推荐条件生成小红书搜索关键词。用户可以查看、启用、停用或补充关键词。

用户从产品页面点击“搜索小红书机会”后，API 调用 `/api/v1/feeds/search`。搜索结果保存笔记标题、正文摘要、作者、互动数据、发布时间、`feed_id` 和 `xsec_token`。对高相关笔记进一步调用笔记详情接口，读取前十条一级评论及必要的子评论。

系统识别两种机会：

- `NOTE_COMMENT`：直接对笔记发表一级评论。
- `COMMENT_REPLY`：回复笔记下某条明确的问题、需求或求助评论。

机会评分由产品相关性、用户意图、时效、互动质量、是否已处理和风险信号组成。相同产品、笔记和目标评论必须幂等，重复搜索不会重复创建机会。

## 草稿与人工确认

LLM 根据 Product Brain 证据、笔记上下文和目标评论生成简短、自然、有帮助的中文草稿。草稿必须明确产品关系，不编造功能或用户成果，不主动索取私信，不使用重复模板。后端限制长度、敏感表达和重复度。

每条机会显示目标笔记、作者、原始内容、目标评论、评分依据和草稿。用户可以重新生成或编辑草稿。

真实执行必须点击“确认并评论”或“确认并回复”。确认弹窗完整展示：当前小红书账号、目标笔记、目标用户、操作类型和最终文本。前端先请求一次性确认令牌，令牌绑定产品、机会、草稿版本、账号和操作类型，短时间内有效且只能使用一次。后端校验令牌后才调用 MCP 评论接口。

默认每日产品操作上限为 3，账号级操作间隔至少 60 秒。同一笔记只允许一次未关闭的评论动作。失败不消耗幂等键，成功请求重复提交只返回已有发布记录。

## 对话跟进

发布成功后创建对话记录。用户可以主动刷新目标笔记详情，系统匹配当前账号评论下的新回复并更新对话状态。系统可以生成回复草稿，但每一次回复仍需人工确认。达到最大跟进次数、用户拒绝、内容删除或用户手动停止后关闭对话。

不在后台高频轮询小红书；刷新由用户主动触发，后续可增加低频调度但不属于本次范围。

## 页面结构与文案

主导航调整为：

- 总览
- 产品
- 机会发现
- 评论草稿
- 对话跟进
- 小红书账号

产品切换器、排序、七天回收站和 Product Brain 保留。原“候选社区”“社区规则”“自动发布”“安全模拟”和 Reddit 审批状态全部删除或替换。总览展示小红书登录状态、今日搜索数、待确认草稿、已评论、收到回复和异常数。

## 错误处理

- MCP 未启动：显示“小红书服务未启动”与启动指引。
- 未登录或 Cookie 失效：停止搜索和评论，展示扫码入口。
- 二维码过期：允许刷新，不保留旧二维码。
- 搜索限流或页面结构变化：保存错误原因，不创建空机会。
- 评论执行结果不确定：不自动重试写操作，标记“需要人工核对”。
- 登录账号发生变化：所有旧确认令牌立即失效。

## 测试与验收

- 客户端契约测试覆盖健康、登录、二维码、搜索、详情、评论和回复的成功与错误响应。
- API 测试覆盖搜索导入、幂等、机会分类、草稿、一次性确认、操作限额、评论发布、回复发布和写操作不自动重试。
- 前端测试覆盖账号状态、二维码、机会筛选、草稿编辑和确认弹窗。
- 浏览器实测完成扫码登录、真实搜索、笔记详情导入和生成草稿。
- 真实评论只使用用户明确选中的测试机会，并在最终点击前再次请求用户确认；未获得该次确认时只验证到草稿阶段。
- Ruff、Pytest、前端测试、ESLint、TypeScript、Next.js 生产构建和 Docker 健康检查全部通过。
